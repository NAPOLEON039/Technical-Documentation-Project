CSS selectors documentation

Type selectors

HTML elements are written using a tag with some attributes like "class" and "id" which are quite common for nearly all elements. We can say that the type of an element is denoted by it's tag. Here are a few examples

  1. <a href="#">Follow on Twitter<a>
    <a href="#">Send an email<a>
    <a href="#">Follow on Reddit<a>

    Now to select all the links above, we need to target them using their type. We can do this using the tag.

    a {
      background-color: green;
    }

    All links will get a background color of green since we have targeted them by type.

  2. <p>This is a paragraph<p>
    <div>This is a div<div>

    Here, if we use the type selector "p", it will only target the first element and not the second, because they are both of different types.

first-child and last-child

:first-child and :last-child are pseudo-classes that make it posible to select the first and last child of a particular element. So if an element has 3 child elements, we can use :first-child to select only the very first child from the 3 child elements.

Similarly, using :last-child will select the very last child element from those 3 elements. Here is an example:

HTML:
<ul>
  <li> Number 1 <li>
  <li> Number 2 <li>
  <li> Number 3 <li>
  <li> Number 4 <li>
  <li> Number 5 <li>
<ul>
CSS:
li:first-child {
  color: red;
}

In this example we apply the pseudo-class :first-child to a list element. The result would be:

  • Number 1
  • Number 2
  • Number 3
  • Number 4
  • Number 5

not pseudo-class

The :not() pseudo-class is perhaps the most powerful of all pseudo-classes. It returns all elements that do not match the selection.

For example, we need to select link elements but leave out those that have the class of "visited". In that case, the CSS would be something like this:

a:not(.visited) {
  color: green;
}

This will target all the links that do not have the class of "visited" and give them a font color of green

Selecting specific elements

:first-child and :last-child allow us to select the first and last child of an element. But what if we want to select another child besides the first or the last? For that we have :nth-child()

Using :nth-child() we can specify a particular number and that child will be selected. Here is an example:

HTML:
<ul>
  <li> Number 1 <li>
  <li> Number 2 <li>
  <li> Number 3 <li>
  <li> Number 4 <li>
  <li> Number 5 <li>
<ul>
CSS:
li:nth-child(4) {
  color: blue;
}

And this is the result:

  • Number 1
  • Number 2
  • Number 3
  • Number 4
  • Number 5

We can also select even or odd number of elements. Using the same example as above, we'll make changes to color all even elements only.

CSS:
li:nth-child(even) {
  color: blue;
}
Result:
  • Number 1
  • Number 2
  • Number 3
  • Number 4
  • Number 5

Calculating specificity of CSS

CSS stands for Cascading Style Sheets. We specify rules that are used to apply effects to HTML elements. We can target an element or a group of elements and apply styles to them. If we target multiple elements, then the style that is specified most recently is applied. A similar example would be: if we keep a red-colored book on a table and then keep a green-colored one directly on top of the red book, we'd see only the green book from above since the red one was kept before it and is hidden beneath it. In the same way, the most recent style rule is applied when two rules target the same element. This is the cascading feature of CSS.

But before this, the specificity of the rule is checked. This means that the rule which targets an element more specifically, will get precedence. Only when all rules targeting the same element have equal specificity that cascading is applied.

Calculating the specificity of a CSS rule is simple to remember. Specificity is denoted by 3 digits - 0,0,0. Counting the number of id selectors gives us the value of the first digit. So if we have a CSS rule like this - 

#code > #display {..}
then the specificity of this rule is 2,0,0.
The next digit is calculated by counting the number of class selectors, attribute selectors and pseudo-classes. If there are multiple of them they simply add up. Let's see an example:

CSS:
#heading .examples:first-child {....}

Here, we have one id selector (#heading), one class selector (.examples) and one pseudo-class (:first-child). That means the specificity of this CSS rule is 1,2,0. Now, for the final digit, we count the number of type selectors and pseudo-elements. Now let's see an example to understand how to determine which rule will be applied when two rules target the same element.

Let's take an element with the class of book. Two CSS rules will try to apply color to it. The rule with the higher specificity will be used.

div#materials > .book {
  color: orange;
}
#materials > .book {
  color: blue;
}

In the first rule, we have one type selector (div), one id selector (#materials) and finally one class selector (.book). The specificity of the first rule is 1,1,1.
The second rule has one id selector (#materials) and one class selector (.book) so it's specificity becomes 1,1,0.

This means that even though the second rule should be applied because of cascading, since CSS first checks the specificity of the rules, the first one will be applied and the element will have a color of orange.
